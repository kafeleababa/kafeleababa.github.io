Control Flow and Loops

The Control Flow refers to the direction that code is read and subsequently written, which is left-to-right, top-to-bottom. This means the layout and structure of the code are crucial to having it run smooth and efficiently. Using Folding Laundry as an example of a flow. Pick up item, Fold item, Put item away. This is how a program will read and then execute the code.

A loop is best described as a set of instructions that the program will follow until the conditions are met or the loops itself ends. To use the folding laundry analogy again, a loop would look like this: Pick up item, Fold item, Put item away, Return to basket. Repeat until the basket is empty.

Functions

A function is a chuck of code which, when run or 'called', will execute that code.

Function naming follows the same rules as variables, multiple words are written in camelCase, and may contain letters, numbers, dollar signs or underscores. Within the function you can set parameters (p1, p2) which, when this particular function is called will return the sum of p1 * p2.

The DOM

The DOM or Document Object Model, is an interface used to represent a webpage and its contents. The DOM structures your page into a hierarchy style tree that displays all of the different parts or nodes which make up the page. Each node is displayed as an object which represents an HTML Element. Each of these objects can be found and then altered within the DOM using the DevTools interface in most browsers.
